## 一、项目概述

### 1.1 一句话定义

**轻量级在线考试 SaaS** — 一个专门做"出题→考试→改卷→分析"闭环的小而精产品，对标超星学习通/雨课堂的考试模块，但更轻、更便宜、体验更好。

### 1.2 目标用户

| 用户 | 需求 | 规模预估 |
|------|------|---------|
| 中小培训机构 | 阶段性测验、模拟考试、成绩分析 | 每个机构 100-1000 学生 |
| K12 学校班级 | 课堂小测、周考月考、错题重练 | 每个班级 30-60 学生 |
| 企业内部培训 | 入职考核、技能认证、员工考试 | 每个企业 50-2000 员工 |

### 1.3 核心价值（为什么有人愿意付钱）

| 现状（超星/雨课堂） | 我们能做的 |
|-------------------|-----------|
| 学校买整套系统，一年几十万，功能冗余 | 按学生数/年费，小机构一年几百到几千 |
| 部署复杂，需要 IT 支持 | 微信扫码即用，零部署成本 |
| 数据导出繁琐 | 实时报表、Excel 一键导出 |
| 移动端体验差 | H5 移动端即开即用，微信内置浏览器直接打开 |

### 1.4 商业模型

```
免费版：100 学生以内，基础题库 + 考试功能（拉新）
Pro 版：按学生数阶梯定价，¥199/100人/年（核心收入）
企业版：私有部署 + 定制，按年签合同（高客单价）
```

## 二、技术架构设计

### 2.1 整体拓扑

```
┌─────────────────────────────────────────────────────┐
│                    用户侧                             │
│  Web 管理后台（教师）    H5 移动端（学生）             │
└──────────────────┬──────────────────┬───────────────┘
                   │ HTTPS/API        │ 微信云调用
                   ▼                  ▼
┌─────────────────────────────────────────────────────┐
│              Nginx（反向代理 + SSL 终结）              │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│              API Gateway / 负载均衡                   │
├──────────────────┬──────────────────┬───────────────┤
│  NestJS 后端     │  WebSocket        │ 静态资源 CDN  │
│  REST API        │  实时通知         │               │
└──────┬───────────┴────────┬─────────┴───────────────┘
       │                    │
┌──────▼──────┐    ┌───────▼────────┐
│ PostgreSQL   │    │  Redis         │
│ 主库 + 只读   │    │  缓存/会话/队  │
│ 迁移：Prisma  │    │  列/实时状态   │
└─────────────┘    └────────────────┘
```

### 2.2 技术选型及理由

| 层 | 技术 | 理由 |
|----|------|------|
| 后端框架 | NestJS | 你已验证过（WZRY Gamer），TypeScript 全栈一致 |
| 前端（教师） | Next.js + Tailwind | 和 WZRY 一致的栈，SSR 利于 SEO |
| 前端（学生） | H5 移动端（响应式，微信内置浏览器可用） | 零门槛，无需审核资质，V2 再加小程序 |
| 数据库 | PostgreSQL + Prisma | 你已验证过，类型安全迁移 |
| 缓存 | Redis | 考试实时状态、防作弊心跳检测 |
| 部署 | Docker Compose | 你已验证过，满足早期规模 |
| CI/CD | GitHub Actions | 你已验证过 |
| 监控 | Sentry（错误追踪）| 最少配置，最大收益 |

### 2.3 核心模块

```
考试 SaaS
├── 租户管理模块（Multi-Tenant）
│   ├── 租户注册/登录/配置
│   ├── 按租户隔离数据（tenant_id 分片）
│   ├── 套餐管理（免费/Pro/企业）
│   └── 子账号（教师/管理员 RBAC）
│
├── 题库管理模块
│   ├── 题目 CRUD（单选/多选/判断/填空）
│   ├── 题目分类/标签/难度
│   ├── 批量导入（Excel）——Word V1 暂不支持（格式过于多样化）
│   └── 题库共享/复制
│
├── 考试管理模块
│   ├── 组卷（手动选题/随机抽题/智能组卷）
│   ├── 考试配置（时间/防作弊/及格线）
│   ├── 发布/分发（二维码/链接/班级码）
│   └── 监考（进入记录/切屏提醒/强制交卷）
│
├── 考试执行模块（学生端 — H5 移动端）
│   ├── V1 不做小程序（审核 1-7 天 + 需企业资质）
│   ├── 改用 H5 响应式，微信内置浏览器直接打开
│   ├── 扫码/链接进入考试（学号/准考证号 + 随机码）
│   ├── 答题（单题/列表/标记回头）
│   ├── 倒计时 + 自动交卷
│   ├── 防作弊：visibilitychange + blur/focus + 心跳上报
│   └── 成绩实时显示
│
├── 成绩分析模块
│   ├── 个人成绩/排名
│   ├── 班级统计（平均分/最高/最低/分布）
│   ├── 题目分析（正确率/选项分布/难度系数）
│   ├── 错题集（自动生成）
│   └── 导出（Excel/PDF）
│
└── 基础设施模块
    ├── Sentry 错误追踪
    ├── 健康检查端点
    ├── 审计日志
    ├── 数据库自动备份
    └── 部署脚本

### 2.4 Redis 在考试场景中的具体角色

| 作用 | 说明 |
|------|------|
| 心跳检测 | 学生端每 10s 上报一次，Redis 记录最后心跳时间，超 30s 无心跳标记异常 |
| 防作弊时间片 | 切出考试页面时记录离场时间戳到 Redis，累计超限标记作弊 |
| 考试实时人数 | Hash Set 记录在线考生，教师端可查看 |
| 提交缓冲 | 考试结束时大量交卷 → Redis 队列缓冲，防 DB 瞬间压力 |
| 答题缓存 | 中途数据缓存至 Redis，防断网丢题 |

```

## 三、安全设计（基于 OWASP API Top 10）

| 风险 | 应对策略 | 优先级 |
|------|---------|-------|
| 租户间数据泄露 | tenant_id 全局过滤器，所有查询强制追加租户条件 | P0 |
| 考试作弊 | 切屏检测 + 心跳 + 答题时间记录 + 随机选项顺序 | P0 |
| JWT 未授权访问 | Access + Refresh Token，7天轮换 | P0 |
| SQL 注入 | Prisma 参数化查询，禁止 raw query | P0 |
| XSS/CSRF | NestJS Helmet + 输入校验 class-validator | P1 |
| 成绩篡改 | 所有提交记录服务器时间戳，客户端仅读 | P1 |
| 接口限流 | Redis + @nestjs/throttler，按租户限流 | P1 |
| 敏感数据 | 成绩导出只含必要字段，不泄露学生个人信息 | P2 |

## 四、质量门禁（基于 Google 审查标准 + Release Gate）

### 4.1 开发流程（Huang-bao-sheng 覆盖）

```
Phase 1: 需求梳理 → 设计文档
Phase 2: 任务规划 → task_plan.md
Phase 3: 实施计划 → 拆到最小可执行步骤

Release Gate: 安全 / 性能 / a11y / 文档 / 迁移
```

### 4.2 测试策略

| 层级 | 工具 | 覆盖率目标 | 执行时机 |
|------|------|-----------|---------|
| 单元测试 | Jest (NestJS) | ≥80% | 每次提交 |
| 集成测试 | Supertest | 关键 API 路径 | PR 合并前 |
| E2E（Web） | Playwright | 核心用户路径 | 发布前 |
| 移动端 H5 测试 | 手机浏览器手动走查 | 核心体验路径 | 发布前 |

### 4.3 发布门禁

```
[ ] 全部测试通过
[ ] 无 P0/P1 安全漏洞
[ ] Sentry 无新增 error
[ ] 数据库迁移脚本已验证（回滚可执行）

[ ] 性能基线已记录（后续版本可对比）
```

## 五、数据模型（V1 核心表）

```
Tenant             — 租户
├── id / name / plan / created_at
│
User               — 用户（教师/管理员）
├── id / tenant_id / role / name / email / password_hash
│
Student            — 学生（属于租户）
├── id / tenant_id / name / class / wechat_openid
│
│ 核心实体均支持软删除（deleted_at TIMESTAMP NULL）：
│   受影响表：QuestionBank / Question / Exam / ExamRecord / Student
│   查询强制加 WHERE deleted_at IS NULL，Prisma 中间件自动追加
│   防止教师误删后无法恢复
│
QuestionBank       — 题库
├── id / tenant_id / name / description
│
Question           — 题目
├── id / bank_id / type(单选/多选/判断) / content / options
│   / answer / difficulty / tags
│
Exam               — 考试
├── id / tenant_id / title / description / time_limit
│   / pass_score / anti_cheat / status / created_by
│
ExamQuestion       — 考试题（关联表，含分值 + 序号）
├── exam_id / question_id / score / sort_order
│
ExamRecord         — 考试记录（学生答题）
├── id / exam_id / student_id / start_time / end_time
│   / score / status(进行中/已交卷/被取消)
│
Answer             — 答题详情
├── record_id / question_id / selected_answer / correct
│   / time_spent
```

## 六、路标规划

### V1（MVP，6-8 周）

**核心交付：** 一套完整的"出题→考试→改卷"闭环

| 阶段 | 内容 | 依赖 |
|------|------|------|
| Week 1-2 | 项目骨架搭建（Docker + CI/CD + 数据库 + Auth） | 无 |
| Week 2-3 | 租户 + 用户系统（注册/登录/RBAC） | 骨架 |
| Week 3-4 | 题库 CRUD + Excel 导入 | 用户系统 |
| Week 4-5 | 考试管理（组卷/发布/分发二维码） | 题库 |
| Week 5-6 | H5 学生端（扫码进考/答题/交卷/看成绩） | 考试管理 |
| Week 6-7 | 成绩分析 + 报表 + 导出 | 考试数据 |
| Week 7-8 | 防作弊 + 安全审计 + Release Gate | 全部功能 |

### V2（验证后迭代，2-4 周）

- 智能组卷（根据错题率自动调整难度）
- 多班级管理
- 学生错题本自动生成
- WebSocket 实时监考面板

### V3（商业扩展）

- 支付集成（微信支付）
- 套餐管理（免费/Pro/企业）
- 私有部署方案
- 考试成绩分析 AI 报告

| 面试问题 | 你能讲的故事 |
|---------|-----------|

| "一次线上问题排查" | 把 WZRY 的 axios baseURL 不对 / 登录 token 字段不匹配拿来改一改 |

- **教学管理系统**（考勤 + 作业 + 课件）
- **竞赛平台**（公开考试、排行榜）
- **证书考试系统**（对接人社/行业协会）

每一个扩展都是一个新入口、一个新收费点。
