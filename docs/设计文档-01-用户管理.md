# 用户管理模块 设计文档

> 版本: V1.0 | 日期: 2026-05-28 | 状态: 设计完毕

## 1. 概述

ADMIN 管理租户内教师账号：创建（系统生成临时密码）、列表（分页+搜索）、启用/禁用。
教师首次登录后强制改密，禁用账号无法登录。

## 2. 数据模型

### User 新增字段

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| mustChangePassword | Boolean | false | 首次登录后强制改密 |
| disabledAt | DateTime? | null | 非 null = 已禁用 |

已有字段 (Day 0 Auth): id, tenantId, role, name, email, passwordHash, createdAt, updatedAt

### Prisma Schema 增量

```prisma
model User {
  // ... existing fields ...
  mustChangePassword Boolean  @default(false) @map("must_change_password")
  disabledAt         DateTime? @map("disabled_at")
}
```

## 3. API 设计

### POST /api/v1/tenant/users — 创建教师

```
Auth: JWT + @Roles(ADMIN)
Body:
{
  "name": "李四",
  "email": "lisi@example.com"
}

Response (201):
{
  "id": "...",
  "name": "李四",
  "email": "lisi@example.com",
  "role": "TEACHER",
  "tempPassword": "aB3xK9mQ"   // 仅此一次可见
}
```

规则:
- 系统生成 8 位随机密码 (大小写+数字)
- bcrypt 哈希存入 passwordHash
- mustChangePassword = true
- 响应仅在 201 返回 tempPassword，不存库

### GET /api/v1/tenant/users — 用户列表

```
Auth: JWT + @Roles(ADMIN)
Query: ?page=1&limit=20&search=张&role=TEACHER

Response (200):
{
  "data": [
    {
      "id": "...",
      "name": "张三",
      "email": "zhangsan@example.com",
      "role": "ADMIN",
      "mustChangePassword": false,
      "disabledAt": null,
      "createdAt": "2026-05-28T00:00:00Z"
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 20
}
```

规则:
- 自动过滤本租户 (req.tenant.id)
- 不返回 passwordHash
- search 模糊匹配 name 或 email
- role 精确匹配

### PATCH /api/v1/tenant/users/:id — 启用/禁用

```
Auth: JWT + @Roles(ADMIN)
Body: { "action": "enable" | "disable" }

Response (200):
{
  "id": "...",
  "disabledAt": null   // enable → null, disable → now()
}
```

规则:
- 不能禁用自己 (id === req.user.sub → 403)
- 不能禁用其他 ADMIN (可选，V1 不强制)
- disabledAt = null 表示启用，disabledAt = new Date() 表示禁用

### POST /api/v1/auth/set-password — 首次改密

```
Auth: Temp JWT (type: "set-password")
Body: { "password": "NewPass123" }

Response (200):
{
  "accessToken": "...",
  "refreshToken": "...",
  "user": { "id": "...", "name": "...", "role": "TEACHER" }
}
```

规则:
- Temp JWT 不含 tenantId/role → TenantGuard 跳过注入，RolesGuard 无匹配放行
- 验证通过后设 passwordHash + mustChangePassword=false + 签发正式双 Token

## 4. 登录检查顺序

AuthService.login 执行顺序：

```
1. bcrypt.compare(dto.password, user.passwordHash)  → 密码错误 401
2. user.disabledAt !== null                          → 403 账号已禁用
3. user.mustChangePassword === true                  → 签发 Temp JWT (type: "set-password")
4. 其他情况                                          → 签发正式 Token
```

Temp JWT payload: `{ sub: userId, type: "set-password" }`
- 不含 tenantId / role
- 过期时间: 30 分钟
- 只能调 POST /auth/set-password

## 5. 安全措施

| 措施 | 实现 |
|------|------|
| 禁用拦截 | login 阶段检查 disabledAt，不在 Guard 层查 DB |
| 改密强制 | mustChangePassword=true → 签发 Temp JWT，不含 tenantId/role |
| Temp 密码 | 8 位随机 (crypto.randomBytes + 大小写+数字)，仅 201 响应可见 |
| 密码不存库 | tempPassword 仅在响应中返回，不落任何表 |
| 租户隔离 | GET 列表自动追加 tenant_id，列表仅本租户可见 |
| 防自禁 | PATCH 禁用自己返回 403 |

## 6. 不做 (V1 范围外)

- 审批流程 (教师注册需 ADMIN 审批)
- 批量导入教师
- 角色切换 ADMIN↔TEACHER
- 密码重置/忘记密码
- 邮箱通知 (V1 不发送任何邮件)
